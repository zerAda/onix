## Naming Conventions

**Files:**
- `main.py` : Entry point (FastAPI app)
- `config.py` : Settings / configuration
- `<component>.py` : Specific functionality (e.g., `cache.py`, `audit.py`, `guardrail.py`)
- `test_*.py` : Unit tests (collocated with source)
- `*.example.json` : Configuration examples
- `conftest.py` : Pytest fixtures / shared test config
- `requirements.txt` : Runtime deps ; `requirements-dev.txt` : dev deps

**Directories:**
- `app/` : Application code (core logic)
- `config/` : Configuration files (examples, static data)
- `tests/` : Test suite (unit + integration + e2e)
- `docs/` : Documentation
- `audit-reality/` : Audit gap tracking

- `monitoring/` : Observability stack
- `branding/` : Brand assets (nginx overlay)
- `scripts/` : Operational scripts
- `deploy/k8s/`, `deploy/azure/` : Deployment targets

**Environment Variables:**
- Uppercase, underscore-separated (e.g., `POSTGRES_HOST`, `ENCRYPTION_KEY_SECRET`)
- Prefixed by component if applicable (e.g., `GRAFANA_ADMIN_USER`, `FABRIC_CLIENT_ID`)
- `:?` in compose (mandatory) or `:-` (optional with default)
- Never in git (`.env` gitignorГ©)

**Test Files:**
- `test_<module>.py` : Unit tests for `<module>.py`
- `test_integration_<scenario>.py` : Integration tests (cacheв†”ACL, streaming, etc.)

## Where to Add New Code

**New Feature (within existing scope):**
- **Primary code** : `access-gateway/app/<component>.py` (if gateway) OR `actions/app/<component>.py` (if actions)
- **Tests** : `access-gateway/tests/test_<component>.py` or `actions/tests/test_<component>.py`
- **Config** : Update `config.py` with new Settings fields ; add `.env.template` entries
- **Docs** : Update `docs/audit-reality/<scope>.md` (mark gap вњ…) ; add proof file:line

**New Microservice / Autonomous Component:**
- **Directory** : Create `<service>/` at root (e.g., `monitoring/` for obs stack)
- **Entry** : `<service>/app/main.py` or `<service>/docker-compose.yml` (if standalone stack)
- **Docs** : `docs/<service>.md` ; add to `DOCS_INDEX.md`
- **Ralph scope** : Create `ralph/scopes/<service>.md` + `ralph/state/<service>.md` (if needs orchestration)

**New Source of Truth / Data Feed:**
- **ACL source** : Extend `access-gateway/app/doc_acl.py` (CompositeDocACL, inherit `DocACL`) ; add tests
- **Audit event** : Extend `actions/app/audit_log.py` (new event type + HMAC chain)
- **Observable metric** : Add counter/gauge in component's `metrics.py` ; add Prometheus rule in `monitoring/prometheus/rules/`

**Monitoring / Dashboard / Alert:**
- **Metrics** : Add counter/gauge in component's `metrics.py` ; register in main app
- **Alert rule** : Add to `monitoring/prometheus/rules/onix-alerts.yml` ; test via `make compose-validate`
- **Dashboard** : Add panel to `monitoring/grafana/dashboards/onix-<scope>.json`
- **SLO rule** : Add to `monitoring/prometheus/rules/onix-slo.yml`

**Infrastructure / Deployment:**
- **Local dev** : Update `docker-compose.yml` (service, env, volumes, network)
- **Kubernetes** : Update `deploy/k8s/onix-ha/templates/`
- **Azure IaC** : Update `deploy/azure/bicep/modules/` ; test via `make bicep-validate`

## Special Directories

- `db_volume` : Postgres persistent data

- `ralph/state/logs/*.log` : Timestamped, one per iteration ; not committed (build artifact)
